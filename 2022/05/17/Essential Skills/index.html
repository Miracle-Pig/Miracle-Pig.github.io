类图

  • 统一建模语言(Unified Modeling Language，UML) 可分类成
    • 功能模型：从用户的角度展示系统的功能，包括用例图。
    • 对象模型：采用对象，属性，操作，关联等概念展示系统的结构和基础，包括类别图、对象图。
    • 动态模型：展现系统的内部行为。包括序列图，活动图，状态图。
  • 类图
    • 类的基础属性
      • -： private
      • +： public
      • ~： default
      • #： protected
      • 下划线： static
      • 斜体： 抽象 (注意也可以用两个尖括号包裹来表示抽象，比如 —— <<我是抽象类or接口>>)
      • 冒号前是方法名/变量名（根据有无括号区分），冒号后是返回参数/变量类型（根据有无括号区分），如果没有冒号的话表示方法返回空（也有人通过：void表示返空）
    • 关系

      • 泛化（继承）关系

        • 泛化关系为 is-a 的关系；两个对象之间如果可以用 is-a 来表示，就是泛化关系。
        • 空心三角 + 实线 == 继承关系
        • Markdown：B继承A，B –|> A

      • 实现关系

        • 程序里面实现关系表现为继承抽象类。继承抽象类和实现接口都属于实现关系。
        • 空心三角 + 虚线 == 实现关系
        • Markdown：B实现A，B ..|> A

      • 关联关系

        • 通常用一条直线表示，当然如果需要标明方向可以添加箭头。它是描书不同的类对象之间的关系，通常不会随着状态的变化而变化，可以理解为被关联者属于关联者的一部分。比如A类中的成员变量有B类对象，码农拥有手机
        • 尖括号 + 实线 == 关联关系
        • Markdown：B关联A，B –> A

      • 组合关系（强依赖）/聚合关系（弱依赖）

        • 表示整体由部分构成的关系，不同之处在于：组合关系的整体和部分是强依赖关系，如果整体不存在了，部分也不复存在，如公司由部门组合；聚合关系整体和部分不是强依赖的，即使整体不存在了，部分仍然存在。
        • 实心棱形 + 实线 == 组合关系； 空心棱形 + 实线 == 聚合关系
        • Markdown：B组合A，B *– A；B聚合A，o–A

      • 依赖关系

        • 描述一个对象在运行期间会用到另一个对象的关系。当一个对量的构造方法中依赖另一个对象来构造自己的实例时，或者一个对象依赖另一个对象提供的服务时，这就是依赖关系，一般指的构造方法较多，比如构造方法的入参就是这个类需要依赖过来实例化的对象。
        • 尖括号 + 虚线 == 依赖关系
        • Markdown：B依赖A，B ..> A
